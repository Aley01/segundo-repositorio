SEGUNDO EJERCICIO DE GIT Y GITHUB

¿Que es GIT?

Sistema de control de versiones (archivos). Controlar los cambios de un repositorio. Base de comandos que nos permiten gestionar los cambios.

¿Que es un Repositorio?

Es una carpeta que contiene archivos (proyectos). Biblioteca de códigos y tenerlos guardados y para guardar los cambios.

Guardar los proyectos

¿Que es GITHUB?

Plataforma en internet que se basa en Git, ver gráficamente lo que está sucediento con nuestro repositorio (commits, branchs, versiones, archivos, código de cada archivo) para compartirlo con otros usuuarios.

Github nos genera un perfil para cada usuario con un nombre de usurio.

Comandos de GIT

Para ejecutar los comandos, necesito estar ubicado en el terminal o consola.

  • Git

    Base para ejecutar comandos de Git.

  • Git init

    Inicializar un repositorio de github en la carpeta o ruta actual

  • Git add [archivos a agregar]

    Agregar cambios a mi repositorio, si pongo * agrega todos los cambios actuales.

  • Git commit -m [mensaje que indica los cambios efectuados]

    Confirmar los cambios en el repositorio (Esto genera a nivel interno en git una rama o Branch)

  • Git push origin [rama origen]

    Enviar los cambios confirmados a github.

  • Git log

    Ver el log de todos los cambios que hemos confirmado o commits.

  • Git status

    Ver el estado actual de los archivos en mi repositorio

  • Git branch [nombre de la rama]

    Crear una rama de trabajo a partir de la rama actual.

  • Git checkout [nombre de la rama]

    Ubicamos en la rama para hacer cambios en ella.

  • Git checkout -b

    Crea la rama y se ubica en ella.

  • Git commit --amend

    Agrega cambios al último commit realizado.

  • Git push --force

    Envar cambios al servidor (github) de manera forzada.

    Esto sobreescribre los cambios en la rama.
